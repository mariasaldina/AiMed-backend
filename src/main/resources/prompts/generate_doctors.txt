Тебе дан массив моковых данных медицинских специалистов.

Для каждого специалиста указаны:
- id;
- ФИО;
- массив специальностей;
- стаж работы в годах.

Для каждого специалиста сгенерируй:

- профессиональное описание врача;
- информацию об образовании.

ТРЕБОВАНИЯ К ОПИСАНИЮ

1. От 3 до 10 предложений.
2. Описание должно быть строго профессиональным профилем врача для медицинского портала, рассчитанным на пациентов.
3. Учитывай специальности и стаж врача.
4. Специализации описываются отдельно. НЕ объединяй их в новые гибридные направления и не создавай новых сущностей.
5. Связь между специализациями допускается только если она естественная (например, терапия и кардиология через сердечно-сосудистые заболевания). Не придумывай искусственные связи.
6. Описания разных специалистов должны существенно отличаться друг от друга по стилю и содержанию.
7. Не используй шаблонные фразы вроде:
   "индивидуальный подход к каждому пациенту",
   "постоянно повышает квалификацию",
   "использует современные методы диагностики",
   если они не нужны по контексту.
8. Не упоминай конкретные адреса, телефоны и названия клиник.
9. Стиль: нейтральный, клинический, без эмоциональной окраски.

ТРЕБОВАНИЯ К ОБРАЗОВАНИЮ

1. Для каждой специальности укажи отдельную строку.
2. Каждая строка должна содержать:
   - название российского медицинского вуза;
   - специальность;
   - год окончания.
3. Образование должно выглядеть реалистично и соответствовать специальностям врача.
4. Годы обучения должны быть согласованы со стажем врача.

ОБЩИЕ ПРАВИЛА

1. Описание и образование заполняется ИСКЛЮЧИТЕЛЬНО на русском языке.
2. Все данные должны соответствовать реалиям РФ.
3. Используй разные медицинские вузы из разных регионов России.
4. Не используй Markdown.
5. Не добавляй комментарии и пояснения.
